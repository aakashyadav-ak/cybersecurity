**Penetration Testing Definition:**
A legally authorized, simulated cyber attack on a computer system to evaluate its security

**Key Differences:**

| Term | Meaning |
| :--- | :--- |
| **Vulnerability Assessment** | Finding vulnerabilities (scanning only) |
| **Penetration Testing** | Finding + Exploiting vulnerabilities |
| **Red Teaming** | Full adversary simulation (stealth focused) |

#### Penetration Testing Phases:
```
1. Pre-Engagement → 2. Reconnaissance → 3. Scanning → 4. Exploitation → 5. Post-Exploitation → 6. Reporting
```


___
# Types of Penetration Testing & Areas

| Type | Knowledge Level | Real-World Scenario |
| :--- | :--- | :--- |
| **Black Box** | Zero knowledge | External attacker simulation |
| **White Box** | Full knowledge (source code, network diagrams) | Internal security audit |
| **Gray Box** | Partial knowledge (credentials, some info) | Insider threat simulation |
