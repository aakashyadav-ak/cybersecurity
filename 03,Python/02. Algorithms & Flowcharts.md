# Algorithm 
An algorithm is a step-by-step procedure to solve a problem or accomplish a task.
```
┌─────────────────────────────────────────────────────────────────┐
│                    WHAT IS AN ALGORITHM?                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Algorithm = Recipe for solving a problem                       │
│                                                                  │
│   Characteristics:                                               │
│   ✓ Clear and unambiguous steps                                 │
│   ✓ Has definite start and end                                  │
│   ✓ Produces correct output                                     │
│   ✓ Finite number of steps                                      │
│   ✓ Language independent                                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```


**example:** Algorithm to Add Two Numbers
```
Algorithm: Add_Two_Numbers

Step 1: START
Step 2: Input first number (A)
Step 3: Input second number (B)
Step 4: Calculate SUM = A + B
Step 5: Display SUM
Step 6: STOP
```

**Example:** Algorithm to Check if Number is Even or Odd
```
Algorithm: Check_Even_Odd

Step 1: START
Step 2: Input a number (N)
Step 3: Calculate REMAINDER = N mod 2
Step 4: IF REMAINDER equals 0 THEN
            Display "Number is Even"
        ELSE
            Display "Number is Odd"
Step 5: STOP
```

# Flowchart
A flowchart is a visual/graphical representation of an algorithm.

```
┌─────────────────────────────────────────────────────────────────┐
│                    WHAT IS A FLOWCHART?                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Flowchart = Diagram showing the flow of a program             │
│                                                                  │
│   Purpose:                                                       │
│   • Visualize the logic before coding                           │
│   • Easy to understand and communicate                          │
│   • Helps identify errors in logic                              │
│   • Documentation of the process                                │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Flowchart Symbols
```
┌─────────────────────────────────────────────────────────────────┐
│                    FLOWCHART SYMBOLS                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ╭───────────╮                                                  │
│   │   START   │    TERMINAL (Oval)                              │
│   │   STOP    │    - Start or End of program                    │
│   ╰───────────╯                                                  │
│                                                                  │
│   ┌───────────┐                                                  │
│   │           │    PROCESS (Rectangle)                          │
│   │  x = 5    │    - Calculation or assignment                  │
│   │           │                                                  │
│   └───────────┘                                                  │
│                                                                  │
│   ╱╲                                                             │
│  ╱  ╲                                                            │
│ ╱ x>5 ╲          DECISION (Diamond)                             │
│ ╲  ?  ╱          - Condition check (Yes/No)                     │
│  ╲  ╱                                                            │
│   ╲╱                                                             │
│                                                                  │
│   ╱─────────╲                                                    │
│  ╱           ╲   INPUT/OUTPUT (Parallelogram)                   │
│ ╱  Input x    ╲  - Read input or display output                 │
│ ╲             ╱                                                  │
│  ╲───────────╱                                                   │
│                                                                  │
│       │                                                          │
│       ↓            ARROW (Flow Line)                            │
│       │            - Direction of flow                          │
│                                                                  │
│       ○            CONNECTOR (Circle)                           │
│                    - Connects different parts                   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```
