## Android Studio
Android Studio = Official [^1]IDE for Android development

Built on IntelliJ IDEA
Used to create, debug, and test Android apps

[^1]: An IDE stands for Integrated Development Environment.
	
	An IDE is a tool that helps developers write, test, and debug code efficiently by combining essential development tools into one program.
	
	It is a software application that provides all the tools developers need to write, test, and debug code in one place.
	What an IDE Includes
	
	Most IDEs provide:
	
	Code editor (where you write code)
	
	Compiler / interpreter
	
	Debugger (find and fix errors)
	
	Build tools (compile, package apps)
	
	Project management (folders, files, structure)
	
	Syntax highlighting (colorful code)
	
	Auto-completion (suggests code as you type)

An SDK stands for Software Development Kit.

It is a collection of tools, libraries, documentation, and resources that developers use to build applications for a specific platform.

Android SDK

When learning Android, SDK usually refers to the Android SDK, which includes:

- ADB (Android Debug Bridge)
- Fastboot
- Android emulator tools

## Native Apps
Native App = Built specifically for Android using Java/Kotlin

Compiled to DEX bytecode
Runs on Android Runtime (ART)

## Native Code (C/C++)
Native Code = C/C++ code compiled to machine code

Uses Android NDK (Native Development Kit)
Compiled to .so (Shared Object) files
Harder to reverse engineer than Java

## JavaScript & WebView
#### WebView
WebView = Browser component embedded in app

Displays web content inside native app
Can execute JavaScript
Bridge between web and native code
