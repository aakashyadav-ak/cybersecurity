# Sets
```
┌─────────────────────────────────────────────────────────────────┐
│                    WHAT IS A SET?                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   A SET is an unordered collection of UNIQUE items.            │
│                                                                  │
│   Think of it like a bag of unique items:                        │
│   • No duplicates allowed                                        │
│   • No specific order                                            │
│   • Fast membership testing                                      │
│                                                                  │
│   Real-world examples:                                           │
│   • Unique IP addresses that accessed a server                  │
│   • Unique ports found during a scan                            │
│   • Unique usernames in a system                                │
│                                                                  │
│   In Python: {1, 2, 3} or {"apple", "banana", "cherry"}         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Set Definition
### Creating Sets
```python
# Sets use curly braces { } like dictionaries
# But contain only values (no key:value pairs)

# Set of integers
ports = {22, 80, 443, 8080}

# Set of strings
protocols = {"HTTP", "HTTPS", "FTP", "SSH"}

# Mixed types (but must be immutable)
mixed = {1, "hello", 3.14, True}

# Check type
print(type(ports))    # <class 'set'>
```

### Empty Set (Important!)
```python
# WRONG way to create empty set
wrong = {}              # This creates an EMPTY DICTIONARY!
print(type(wrong))      # <class 'dict'>

# CORRECT way to create empty set
correct = set()
print(type(correct))    # <class 'set'>
```

### Duplicates Are Automatically Removed
```python
# Sets automatically remove duplicates!

numbers = {1, 2, 2, 3, 3, 3, 4, 4, 4, 4}
print(numbers)    # {1, 2, 3, 4} - Duplicates gone!

# Very useful for finding unique values
ip_logs = {"192.168.1.1", "10.0.0.50", "192.168.1.1", "192.168.1.1", "10.0.0.50"}
print(ip_logs)    # {'192.168.1.1', '10.0.0.50'} - Only unique IPs

# Count of unique items
print(f"Unique IPs: {len(ip_logs)}")    # 2
```

### Creating Sets from Other Types
```python
# Method 1: Direct creation
my_set = {1, 2, 3}

# Method 2: set() constructor from list
my_list = [1, 2, 2, 3, 3, 3]
my_set = set(my_list)
print(my_set)    # {1, 2, 3}

# Method 3: set() from string (each character becomes an item)
my_set = set("hello")
print(my_set)    # {'h', 'e', 'l', 'o'} - Note: only one 'l'!

# Method 4: set() from tuple
my_set = set((1, 2, 3))
print(my_set)    # {1, 2, 3}

# Method 5: set() from range
my_set = set(range(1, 6))
print(my_set)    # {1, 2, 3, 4, 5}

# Method 6: Set comprehension
squares = {x**2 for x in range(1, 6)}
print(squares)   # {1, 4, 9, 16, 25}
```

## CHARACTERISTICS OF SETS
```
┌─────────────────────────────────────────────────────────────────┐
│              CHARACTERISTICS OF SETS                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   1. UNORDERED                                                   │
│      • Items have NO defined order                               │
│      • Order may change                                          │
│      • Cannot access by index (no set[0])                       │
│                                                                  │
│   2. NO DUPLICATES                                               │
│      • Each item appears only once                               │
│      • Duplicates automatically removed                          │
│                                                                  │
│   3. MUTABLE                                                     │
│      • Can add and remove items                                  │
│      • But items themselves must be immutable                   │
│                                                                  │
│   4. UNINDEXED                                                   │
│      • Cannot access items by index                             │
│      • Use loops or 'in' to access                              │
│                                                                  │
│   5. FAST MEMBERSHIP TESTING                                     │
│      • Checking if item exists is very fast O(1)                │
│      • Much faster than lists for large data                    │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

```python
# 1. UNORDERED - No defined order
colors = {"red", "green", "blue"}
print(colors)    # Order may vary! Could be {'green', 'red', 'blue'}

# Run multiple times - order might change
for i in range(3):
    print(set([3, 1, 2]))    # Order not guaranteed

# 2. NO DUPLICATES
numbers = {1, 1, 2, 2, 3, 3}
print(numbers)    # {1, 2, 3}

# 3. MUTABLE - Can add/remove
fruits = {"apple", "banana"}
fruits.add("cherry")
print(fruits)    # {"apple", "banana", "cherry"}

# 4. UNINDEXED - Cannot use index
my_set = {10, 20, 30}
# print(my_set[0])    # ERROR! TypeError: 'set' object is not subscriptable

# 5. FAST MEMBERSHIP TESTING
large_set = set(range(1000000))
print(999999 in large_set)    # True (very fast!)
```

## Adding Elements
### add() Method
```python
# add(item) - Add ONE item to the set

ports = {22, 80}
print(f"Before: {ports}")    # {22, 80}

# Add single item
ports.add(443)
print(f"After: {ports}")     # {22, 80, 443}

# Add another
ports.add(8080)
print(f"After: {ports}")     # {22, 80, 443, 8080}

# Adding duplicate has no effect
ports.add(80)                # 80 already exists
print(f"After: {ports}")     # {22, 80, 443, 8080} - No change!
```
