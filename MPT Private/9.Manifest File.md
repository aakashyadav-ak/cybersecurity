A manifest file is a metadata file that describes the contents, configuration, permissions, and dependencies of a software package, application, or project.

Before the Android OS runs an app, it reads this file to understand:

- Who the app is (Package Name).
- What the app can do (Permissions).
- Which screens and background tasks exist (Components).
- How the app interacts with the system and other apps.

**Location:**
**Source Code:** Found in the root of the src/main/ folder.
**Compiled APK:** Found in the root of the APK file (encoded in binary XML).


## AndroidManifest.xml
```
<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.example.myapp">
    
    <!-- PERMISSIONS - Security Critical -->
    <uses-permission android:name="android.permission.INTERNET"/>
    <uses-permission android:name="android.permission.CAMERA"/>
    <uses-permission android:name="android.permission.READ_CONTACTS"/>
    
    <!-- SDK Requirements -->
    <uses-sdk android:minSdkVersion="21" 
              android:targetSdkVersion="33"/>
    
    <application
        android:allowBackup="true"
        android:debuggable="false"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name">
        
        <!-- Entry Points -->
        <activity android:name=".MainActivity"
                  android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>
        
        <!-- Services & Receivers -->
        <service android:name=".MyService" 
                 android:exported="false"/>
        <receiver android:name=".MyReceiver"
                  android:exported="true"/>
                  
    </application>
</manifest>
```

##### Reading Manifest Files: 
**Security:**

| Area | What to Look For | Risk Level | Priority | OWASP Category |
|------|------------------|------------|----------|----------------|
| Permissions | Excessive/unnecessary permissions | HIGH | P1 | M1-Improper Platform Usage |
| Exported Components | exported="true" on activities/services | HIGH | P1 | M1-Improper Platform Usage |
| Debuggable Flag | debuggable="true" in production | CRITICAL | P0 | M10-Extraneous Functionality |
| Backup Settings | allowBackup="true" | MEDIUM | P2 | M2-Insecure Data Storage |
| Network Security | usesCleartextTraffic="true" | HIGH | P1 | M3-Insecure Communication |
| Dependencies | Outdated/vulnerable libraries | HIGH | P1 | M1-Improper Platform Usage |
| Host Permissions | `<all_urls>` or broad patterns | HIGH | P1 | M1-Improper Platform Usage |

#### MPT Vulnerabilities

| Vulnerability | Detection Method | Severity | Exploitation |
|---------------|------------------|----------|--------------|
| Debuggable Application | `android:debuggable="true"` | CRITICAL | ADB shell access |
| Exported Activities | `android:exported="true"` without permission | HIGH | Intent hijacking |
| Exported Services | `android:exported="true"` on services | HIGH | Service abuse |
| Exported Broadcast Receivers | `android:exported="true"` on receivers | HIGH | Broadcast injection |
| Exported Content Providers | `android:exported="true"` on providers | CRITICAL | Data theft |
| Backup Enabled | `android:allowBackup="true"` | MEDIUM | ADB backup extraction |
| Cleartext Traffic | `android:usesCleartextTraffic="true"` | HIGH | MITM attacks |
| Weak Permissions | `protectionLevel="normal"` on custom perms | MEDIUM | Permission bypass |
| Task Hijacking | `taskAffinity` + `allowTaskReparenting` | HIGH | UI spoofing |
| Deep Link Vulnerabilities | Unvalidated intent-filter schemes | HIGH | URL hijacking |
| Implicit Intents | Intent without explicit component | MEDIUM | Intent interception |
| Old SDK Version | `minSdkVersion < 24` | MEDIUM | Legacy exploits |
