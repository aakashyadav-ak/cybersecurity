# Repeater
- **Purpose:** Regenerates/boosts signal to extend distance.
- **Layer:** OSI Layer 1 (Physical)
- **Security view:** No filtering, no intelligence.
# 1. HUB (Layer 1 - Physical Layer)
A Hub is a basic networking device that connects multiple Ethernet devices together, making them act as a single network segment.

- **Purpose:** Sends data to ALL ports (broadcasts everything).
- **Layer:** OSI Layer 1 (Physical)
- **Key point:** One collision domain.
- **Security view:** Very insecure, easy sniffing.

```
┌─────────────────────────────────────────────────────────────┐
│                         HUB                                  │
│                                                              │
│    PC-A ──────┐                      ┌────── PC-C            │
│               │                      │                       │
│               ▼                      ▼                       │
│            ┌──────────────────────────────┐                  │
│            │     PORT1  PORT2  PORT3  PORT4                  │
│            │       │      │      │      │                    │
│            │       ▼      ▼      ▼      ▼                    │
│            │    ┌──────────────────────────┐                 │
│            │    │    ELECTRICAL BUS        │                 │
│            │    │   (All signals shared)   │                 │
│            │    └──────────────────────────┘                 │
│            └──────────────────────────────┘                  │
│               ▲                      ▲                       │
│               │                      │                       │
│    PC-B ──────┘                      └────── PC-D            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## Working:
- **Broadcasting Device:** When a hub receives data on one port, it floods/broadcasts it to ALL other ports
- **No Intelligence:** Cannot read MAC addresses or make forwarding decisions
- **Half-Duplex:** Only one device can transmit at a time
- **Collision Domain:** All connected devices share ONE collision domain


# 2. BRIDGE

A bridge is a Layer 2 (Data Link Layer) device that connects two or more network segments and filters traffic based on MAC addresses.

- **Purpose:** Connects two LAN segments and filters using MAC.
- **Layer:** OSI Layer 2 (Data Link)
- **Key point:** Reduces collisions.
- **Security view:** Better than hub, but still limited.

```
  Segment A              Segment B
[PC1]--[PC2]---[BRIDGE]---[PC3]--[PC4]
     
Collision Domain 1  |  Collision Domain 2
```

### MAC Table (Store MAC table )

**Frame Forwarding Decision**
```
DECISION FLOWCHART:

Frame arrives on Port 1, Destination = CC
        ↓
Is CC in MAC table?
        ↓
    [YES] → CC on Port 2 → Forward to Port 2 ONLY
        
    [NO] → FLOOD to all ports except source
```

#### Spanning Tree Protocol (STP) - Preventing Loops
```
┌─────────────────────────────────────────────────────────────┐
│              SPANNING TREE PROTOCOL                         │
│                                                             │
│ PROBLEM: Bridge loops cause broadcast storms                │
│                                                             │
│        Bridge A ◄────────────► Bridge B                     │
│            ▲                       ▲                        │
│            │                       │                        │
│            └───────► Bridge C ◄────┘                        │
│                                                             │
│   If broadcast enters → loops forever → NETWORK CRASH!      │
│                                                             │
│ SOLUTION: STP blocks redundant paths                        │
│                                                             │
│        Bridge A ◄────────────► Bridge B (Root Bridge)       │
│            ▲                       ▲                        │
│            │                       │                        │
│            └───── X Bridge C ◄─────┘                        │
│                   (Blocked Port)                            │
│                                                             │
│ STP Process:                                                │
│ 1. Elect Root Bridge (lowest Bridge ID)                     │
│ 2. Calculate shortest path to root                          │
│ 3. Block redundant ports                                    │
│ 4. If failure → unblock backup path                         │
└─────────────────────────────────────────────────────────────┘
```

# 3. SWITCH (Layer 2 - Data Link Layer)
- A Switch is essentially a "Multi-port Bridge." It creates a dedicated path between the sender and receiver.
- A Switch is an advanced multi-port bridge that operates at Layer 2 (or Layer 3 for multilayer switches). It's the backbone of modern LANs, providing dedicated bandwidth to each port.

- **Purpose:** Sends frames only to the correct port using MAC table.
- **Layer:** OSI Layer 2 (Data Link)
- **Key points:**
  - Uses MAC address table (CAM table)
  - Each port is separate collision domain
- **Security view (VAPT):**
  - ARP spoofing possible
  - VLAN hopping possible (misconfig)
  - MAC flooding possible

## Switching Methods
```
┌─────────────────────────────────────────────────────────────┐
│                   SWITCHING METHODS                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. STORE-AND-FORWARD                                        │
│  ════════════════════                                        │
│     ┌─────────────────────────────────────┐                 │
│     │  Receive entire frame               │                 │
│     │         ↓                           │                 │
│     │  Check FCS (error check)            │                 │
│     │         ↓                           │                 │
│     │  If valid → Forward                 │                 │
│     │  If invalid → Drop                  │                 │
│     └─────────────────────────────────────┘                 │
│     ✓ Highest reliability                                    │
│     ✓ No error propagation                                   │
│     ✗ Highest latency                                        │
│                                                              │
│  2. CUT-THROUGH                                              │
│  ═══════════════                                             │
│     ┌─────────────────────────────────────┐                 │
│     │  Read only destination MAC (6 bytes)│                 │
│     │         ↓                           │                 │
│     │  Start forwarding immediately       │                 │
│     └─────────────────────────────────────┘                 │
│     ✓ Lowest latency                                         │
│     ✗ Forwards corrupt frames                                │
│                                                              │
│  3. FRAGMENT-FREE                                            │
│  ═══════════════════                                         │
│     ┌─────────────────────────────────────┐                 │
│     │  Read first 64 bytes                │                 │
│     │  (collision window)                 │                 │
│     │         ↓                           │                 │
│     │  Then forward                       │                 │
│     └─────────────────────────────────────┘                 │
│     ✓ Catches most collision errors                          │
│     ✓ Better latency than store-and-forward                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

#  ==4. ROUTER (Layer 3 Device)==
- A Router is a Layer 3 device that interconnects different networks and makes forwarding decisions based on IP addresses. It's the fundamental device for internetworking and WAN connectivity.
- A Router is a Layer 3 device that forwards packets between DIFFERENT networks using IP addresses and routing tables.

A Router is a network device operating at the Network Layer (Layer 3) that performs packet switching based on logical addressing (IP), maintains routing information bases (RIB), and executes routing protocols to determine optimal paths for data transmission across interconnected networks.

```
│ L = Local        │ Router's own interface IP                │
│ S = Static       │ Manually configured route                │
│ O = OSPF         │ Learned via OSPF protocol                │
│ D = EIGRP        │ Learned via EIGRP protocol               │
│ R = RIP          │ Learned via RIP protocol                 │
│ * = Default      │ Default gateway route                    │
└─────────────────────────────────────────────────────────────┘
```

```
PACKET JOURNEY THROUGH ROUTER:

Step 1: Packet arrives on interface
        ┌─────────────────────────────────────────┐
        │ Ethernet Frame                          │
        │ ┌─────────────────────────────────────┐ │
        │ │ Dest MAC │ Src MAC │ IP Packet     │ │
        │ └─────────────────────────────────────┘ │
        └─────────────────────────────────────────┘

Step 2: Router checks destination MAC
        • If matches router's interface MAC → Accept
        • If doesn't match → Discard

Step 3: Router strips Layer 2 header, examines IP header
        ┌─────────────────────────────────────┐
        │ IP Packet                           │
        │ Src IP: 10.1.1.10                   │
        │ Dst IP: 10.2.2.10  ← Router reads   │
        └─────────────────────────────────────┘

Step 4: Routing table lookup for destination
        Router checks: "How do I reach 10.2.2.10?"

Step 5: Determine exit interface and next-hop

Step 6: Decrement TTL (Time to Live)
        • If TTL = 0 → Discard, send ICMP Time Exceeded
        • If TTL > 0 → Continue

Step 7: Build new Layer 2 header
        • Source MAC = Router's exit interface MAC
        • Dest MAC = Next-hop device MAC (ARP lookup)

Step 8: Forward packet out exit interface
```

## Static vs Dynamic Routing
```
┌─────────────────────────────────────────────────────────────┐
│              STATIC vs DYNAMIC ROUTING                       │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  STATIC ROUTING                                              │
│  ═══════════════                                             │
│  Router(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2│
│  Router(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.1          │
│                                                              │
│  Advantages:                                                 │
│  ✓ No overhead (no routing protocol traffic)                 │
│  ✓ Predictable                                               │
│  ✓ Secure (less attack surface)                              │
│  ✓ Works on low-bandwidth links                              │
│                                                              │
│  Disadvantages:                                              │
│  ✗ Manual configuration required                             │
│  ✗ No automatic failover                                     │
│  ✗ Doesn't scale                                             │
│                                                              │
│  DYNAMIC ROUTING                                             │
│  ═══════════════════                                         │
│  Uses protocols: OSPF, EIGRP, BGP, RIP, IS-IS               │
│                                                              │
│  Router(config)# router ospf 1                               │
│  Router(config-router)# network 10.0.0.0 0.0.0.255 area 0   │
│                                                              │
│  Advantages:                                                 │
│  ✓ Automatic route discovery                                 │
│  ✓ Automatic failover                                        │
│  ✓ Scales well                                               │
│  ✓ Load balancing capable                                    │
│                                                              │
│  Disadvantages:                                              │
│  ✗ CPU/memory overhead                                       │
│  ✗ Bandwidth usage (protocol traffic)                        │
│  ✗ Security concerns (routing attacks)                       │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```
