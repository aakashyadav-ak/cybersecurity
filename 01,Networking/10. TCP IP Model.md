# TCP/IP Model
The TCP/IP Model (Transmission Control Protocol/Internet Protocol) is a practical, four-layer networking framework that forms the foundation of the modern internet.

```
┌─────────────────────────────────────┐
│         APPLICATION LAYER           │  Layer 4
├─────────────────────────────────────┤
│          TRANSPORT LAYER            │  Layer 3
├─────────────────────────────────────┤
│           INTERNET LAYER            │  Layer 2
├─────────────────────────────────────┤
│        NETWORK ACCESS LAYER         │  Layer 1
└─────────────────────────────────────┘
```

#  TCP/IP Model

##  What is the TCP/IP Model?
> **TCP/IP Model** (Transmission Control Protocol/Internet Protocol) is a **conceptual/Practical framework** used to describe network communication. It is the **practical implementation** model used in the **Internet** and most modern networks.

- Also known as: **Internet Protocol Suite** or **DoD Model** (Department of Defense)
- Developed by: **DARPA** in the 1970s
- More **practical** than OSI Model

---

## TCP/IP Model Layers (4 Layers)
```
┌─────────────────────────────────────────────────┐
│ APPLICATION LAYER │ ← Layer 4
│ (HTTP, FTP, SMTP, DNS, DHCP, Telnet, SSH) │
├─────────────────────────────────────────────────┤
│ TRANSPORT LAYER │ ← Layer 3
│ (TCP, UDP, Port Numbers) │

├─────────────────────────────────────────────────┤
│ NETWORK ACCESS LAYER │ ← Layer 1
│ (Ethernet, WiFi, PPP, Frame Relay) │
└─────────────────────────────────────────────────┘
```

#  Detailed Layer Description

## Network Access Layer (Layer 1)

> **Function:** Defines how data is physically transmitted over the network medium

**Responsibilities:**
- Physical addressing (MAC addresses)
- Frame formatting
- Media access control
- Error detection (not correction)
- Physical transmission of bits

**Combines OSI Layers:**
- Physical Layer
- Data Link Layer

#### Protocols & Technologies:
- **Ethernet** (802.3)
- **WiFi** (802.11)
- **PPP** (Point-to-Point Protocol)
- **Frame Relay**
- **ATM** (Asynchronous Transfer Mode)
- **Token Ring**
- **HDLC** (High-Level Data Link Control)

#### Hardware:
- Network Interface Cards (NICs)
- Switches
- Bridges
- Hubs
- Cables (Copper, Fiber)
- Wireless Access Points

#### Data Unit:
- **Frame** (or **Bits** at physical level)

**Network Access Layer Diagram:**
```
┌─────────────────────────────────────┐
│  Ethernet Frame                     │
├──────────┬──────┬─────┬──────┬─────┤
│Preamble  │ MAC  │ MAC │ Data │ FCS │
│   (8)    │ Dest │ Src │      │ (4) │
└──────────┴──────┴─────┴──────┴─────┘

MAC Address: 48-bit (6 bytes)
Example: AA:BB:CC:DD:EE:FF
```

#### Physical Addressing (MAC Addresses)

At Layer 1, devices use MAC addresses to identify each other inside a local network (LAN).

**A MAC address:**
- 48-bit hardware address  
- Assigned to Network Interface Card (NIC)  
- Example:  
  ```
  00:1A:2B:3C:4D:5E
  ```

**When a device sends data in a LAN:**
- It uses **Source MAC**
- It uses **Destination MAC**

Switches forward frames using MAC addresses.

**Example:**
- Your Windows VM sends traffic  
- Switch forwards based on MAC table  

---

#### Frame Formatting

Layer 2 wraps data into a **Frame**.

**Structure of a frame (Ethernet example):**
```
| Destination MAC | Source MAC | Type | Data | FCS |         # This entire structure = Frame
```

**Trailer** = The last field of the frame
**FCS (Frame Check Sequence)** = The content inside the trailer

**Important protocol here:**  
Ethernet

**So:**
- Network layer sends a packet  
- Data Link layer encapsulates it into a frame  
- Adds MAC addresses + trailer  

---

#### Media Access Control (MAC)

If multiple devices share the same medium (like Ethernet cable or Wi-Fi), they need rules to avoid collision.

**Ethernet uses:**
- CSMA/CD (Carrier Sense Multiple Access with Collision Detection)

**Wi-Fi uses:**
- CSMA/CA (Collision Avoidance)

**This ensures:**
- Only one device transmits at a time  
- Reduces collision  

---

#### Error Detection (Not Correction)

Layer 2 detects errors using:

- FCS (Frame Check Sequence)

If a frame gets corrupted:

- Receiver detects error  
- Frame is dropped  
- No correction at this layer  

Correction happens at higher layers (like TCP).

---

#### Physical Transmission of Bits

This is **Layer 1 (Physical Layer)** responsibility.

Layer 1:

- Converts bits into electrical signals (Ethernet)  
- Light pulses (Fiber)  
- Radio waves (Wi-Fi)  

Protocols involved:

- IEEE 802.3  
- IEEE 802.11  

It handles:

- Voltage levels  
- Cable types  
- Bit timing  
- Connectors  

---

## Internet Layer (Layer 2)

> **Function:** Routes packets across networks using logical addressing (IP addresses)
> ==It is responsible for delivering packets across multiple networks.==

**Responsibilities:**
- Logical addressing (IP addresses)
- Routing between networks
- Packet fragmentation and reassembly
- Path determination
- Best effort delivery (unreliable)

**Equivalent OSI Layer:**
- Network Layer

#### Key Protocols:

**IP (Internet Protocol):**
- **IPv4** - 32-bit addressing (e.g., 192.168.1.1)
- **IPv6** - 128-bit addressing (e.g., 2001:db8::1)

**Supporting Protocols:**
- **ICMP** (Internet Control Message Protocol) - ping, traceroute
- **ARP** (Address Resolution Protocol) - Maps IP to MAC
- **RARP** (Reverse ARP) - Maps MAC to IP
- **IPsec** - Security protocol

#### Routing Protocols:
- **RIP** (Routing Information Protocol)
- **OSPF** (Open Shortest Path First)
- **EIGRP** (Enhanced Interior Gateway Routing Protocol)
- **BGP** (Border Gateway Protocol)

#### Hardware:
- Routers
- Layer 3 Switches

#### Data Unit:
- **Packet** (or **Datagram**)

#### 1) Logical Addressing (IP Addresses)

The Internet Layer uses IP addresses to identify devices across networks.

Protocol used:
Internet Protocol (IP)

Example:
```
192.168.1.10
```

Unlike MAC addresses:
- MAC = Physical address (Data Link Layer)
- IP = Logical address (Internet Layer)

Why is it called logical?
- Assigned manually or by DHCP
- Can change
- Not permanently burned into hardware

Purpose:
Identifies devices globally across different networks.

---

#### 2) Routing Between Networks

This layer enables communication between different networks.

Example:

```
Kali VM (192.168.1.20)
        ↓
Router
        ↓
Windows Victim (10.0.0.5)
```

Routers:
- Read destination IP
- Check routing table
- Forward packet to next hop

Routing decisions are based on:
- Destination IP address
- Routing tables

---

#### 3) Packet Fragmentation and Reassembly

Each network has a limit called:

MTU (Maximum Transmission Unit)

If a packet is larger than the MTU:
- It is fragmented into smaller packets
- Each fragment is sent separately
- Destination reassembles them

This process is handled by IP.

---

#### 4) Path Determination

The Internet Layer determines the best path to reach the destination.

Routers use:
- Routing tables
- Metrics (hop count, bandwidth, cost)

Goal:
Find the most efficient route from source to destination.

---

#### 5) Best Effort Delivery (Unreliable)

IP provides best-effort delivery.

**This means:**
- No guarantee packet will arrive
- No guarantee of order
- No guarantee of no duplication
- No guarantee of no corruption

If reliability is required:
It is handled by TCP at the Transport Layer.

IP simply forwards packets without guaranteeing delivery.


---

## Transport Layer (Layer 3)

> **Function:** Provides end-to-end communication, reliability, and flow control

**Responsibilities:**
- End-to-end communication
- Segmentation and reassembly
- Flow control
- Error detection and correction
- Port-based addressing
- Connection management

**Equivalent OSI Layer:**
- Transport Layer

#### Key Protocols:

**TCP (Transmission Control Protocol):**
- **Connection-oriented**
- **Reliable** delivery
- **Ordered** delivery
- **Flow control**
- **Error checking**
- Uses **3-way handshake**
- Use cases: HTTP, FTP, SSH, Email

**UDP (User Datagram Protocol):**
- **Connectionless**
- **Unreliable** (best effort)
- **Faster** than TCP
- **No flow control**
- **Minimal overhead**
- Use cases: DNS, DHCP, VoIP, Video streaming

#### Port Numbers:
- **Well-Known Ports:** 0-1023 (Reserved for common services)
- **Registered Ports:** 1024-49151 (Registered applications)
- **Dynamic/Private Ports:** 49152-65535 (Temporary/client ports)

#### Hardware:
- Firewalls
- Load Balancers

#### Data Unit:
- **Segment** (TCP) or **Datagram** (UDP)


#### TCP 3-Way Handshake:
```
Client                          Server
  |                               |
  |  SYN (Seq=100)               |
  |----------------------------->|
  |                               |
  |  SYN-ACK (Seq=200, Ack=101)  |
  |<-----------------------------|
  |                               |
  |  ACK (Seq=101, Ack=201)      |
  |----------------------------->|
  |                               |
  |  Connection Established       |
  |                               |
```

## Application Layer (Layer 4)
> **Function:** Provides network services directly to end-user applications
> ==The Application Layer is the top layer where the user and applications interact with the network, responsible for data presentation/formatting and for session management between applications.==

**Responsibilities:**
- Application-level protocols
- User interface
- Network process to application
- Data representation
- Session management
- Encryption / Compression / Translation

**Combines OSI Layers:**
- Application Layer
- Presentation Layer
- Session Layer

#### Common Protocols:

**Web & File Transfer:**
- **HTTP** (Hypertext Transfer Protocol) - Port 80
- **HTTPS** (HTTP Secure) - Port 443
- **FTP** (File Transfer Protocol) - Port 20, 21

**Email:**
- **SMTP** (Simple Mail Transfer Protocol) - Port 25
- **POP3** (Post Office Protocol) - Port 110
- **IMAP** (Internet Message Access Protocol) - Port 143

**Network Services:**
- **DNS** (Domain Name System) - Port 53
- **DHCP** (Dynamic Host Configuration Protocol) - Port 67, 68
- **SNMP** (Simple Network Management Protocol) - Port 161, 162

**Remote Access:**
- **Telnet** - Port 23 (Insecure)
- **SSH** (Secure Shell) - Port 22
- **RDP** (Remote Desktop Protocol) - Port 3389

**Other:**
- **SMB** (Server Message Block) - Port 445
- **LDAP** (Lightweight Directory Access Protocol) - Port 389

#### Application Layer Responsibilities

The **Application Layer** provides services directly to user applications and allows them to communicate over the network.

##### 1. Application-level Protocols
These are rules that applications use to communicate on a network.

**Examples**
- HTTP – used for web browsing
- FTP – used for file transfer
- SMTP – used for sending emails

Example: When you open a website, the browser uses **HTTP/HTTPS** to request the webpage.

---

##### 2. User Interface
Allows the **user to interact with the network through applications**.

**Examples**
- Web browsers (Chrome, Firefox)
- Email clients (Outlook, Gmail)
- File transfer programs

Example: When you type `www.example.com` in a browser, you are using the application layer.

---

##### 3. Network Process to Application
Connects **software applications to the network** so they can send and receive data.

Example:
- A browser sending a request to a web server.
- An email client sending a message to a mail server.

Example scenario:
You click **Send Email**, and the application sends the message to the mail server.

---

##### 4. Data Representation
Ensures that **data is in a format both devices can understand**.

Examples:
- Converting text encoding (ASCII, Unicode)
- Formatting images or documents

Example: A file created on Windows can still be opened on Linux or Mac because of standardized data formats.

---

##### 5. Session Management
Responsible for **starting, maintaining, and ending communication sessions** between applications.

Examples:
- Maintaining login sessions on websites
- Keeping a video call connection active

Example: When you stay logged into a website without logging in again.

---

##### 6. Encryption / Compression / Translation

**Encryption**
- Protects data from attackers
- Example: HTTPS encrypts communication between browser and server

**Compression**
- Reduces data size to send faster
- Example: Websites compress images or files before sending

**Translation**
- Converts data into a format different systems can understand
- Example: Converting text encoding formats

---

## Data Encapsulation Process
### Sender

```
Application Layer
DATA
   ↓
Transport Layer
[TCP/UDP Header | DATA]
   ↓
Internet Layer
[IP Header | TCP/UDP Header | DATA]
   ↓
Network Access Layer
[Frame Header | IP Header | TCP/UDP Header | DATA | FCS]
   ↓
Physical Layer
Bits transmitted over the network medium
```

---

### Network Medium

Data travels through the physical network (cables, wireless, etc.)

---

### Receiver

```
Network Access Layer
[Frame Header | IP Header | TCP/UDP Header | DATA | FCS]
(Removes Frame Header and FCS)
   ↓
Internet Layer
[IP Header | TCP/UDP Header | DATA]
(Removes IP Header)
   ↓
Transport Layer
[TCP/UDP Header | DATA]
(Removes TCP/UDP Header)
   ↓
Application Layer
DATA
```

---

### Key Concept
- **Encapsulation:** Each layer adds its own header while sending data.
- **Decapsulation:** Each layer removes its header while receiving data.


### Encapsulation Terms:

| Layer              | Data Unit        | Name                           |
| ------------------ | ---------------- | ------------------------------ |
| **Application**    | Data             | Data / Message                 |
| **Transport**      | Segment/Datagram | Segment (TCP) / Datagram (UDP) |
| **Internet**       | Packet           | Packet / Datagram              |
| **Network Access** | Frame            | Frame                          |
| **Physical**       | Bits             | Bits                           |

## End-to-End Communication Example

## Web Request Flow Through TCP/IP Model

When a user types **www.example.com**, the communication happens in **3 stages**:

DNS Resolution → TCP Handshake → HTTP Request

Each stage travels through **all layers of the TCP/IP model**.

---

## Overall Flow

```
User Request
     │
     ▼
┌───────────────┐
│ 1. DNS QUERY  │
└───────────────┘
     │
     ▼
┌──────────────────┐
│ 2. TCP HANDSHAKE │
└──────────────────┘
     │
     ▼
┌──────────────────┐
│ 3. HTTP REQUEST  │
└──────────────────┘
     │
     ▼
Webpage Response
```

---

# 1️⃣ DNS Query (Find Website IP)

```
Application Layer
DNS Query
"What is the IP of www.example.com?"

        │
        ▼

Transport Layer
UDP Header Added
Port: 53

        │
        ▼

Internet Layer
IP Header Added
Destination: DNS Server

        │
        ▼

Network Access Layer
Ethernet Frame Created
Data Sent Across Network
```

DNS Server responds with:

```
IP Address → 93.184.216.34
```

⬇ Now browser knows **where the website server is**

---

# 2️⃣ TCP 3-Way Handshake (Create Connection)

```
Application Layer
Browser requests connection

        │
        ▼

Transport Layer
TCP Handshake

Client → SYN
Server → SYN-ACK
Client → ACK

        │
        ▼

Internet Layer
IP Packet Sent to Web Server

        │
        ▼

Network Access Layer
Frame transmitted through network
```

⬇ Now **connection between client and server is established**

---

# 3️⃣ HTTP Request (Request Webpage)

```
Application Layer
HTTP Request

GET /index.html
Host: www.example.com

        │
        ▼

Transport Layer
TCP Segment
Port: 80

        │
        ▼

Internet Layer
IP Packet to Server

        │
        ▼

Network Access Layer
Frame sent over network
```

Server responds with:

```
HTML Page
CSS
Images
JavaScript
```

| 1️⃣ DNS Resolution             | 2️⃣ TCP 3-Way Handshake           | 3️⃣ HTTP Request                                              |
| ------------------------------ | --------------------------------- | ------------------------------------------------------------- |
| **Purpose:** Get IP of website | **Purpose:** Establish connection | **Purpose:** Request webpage                                  |
| **Application Layer**          | **Application Layer**             | **Application Layer**                                         |
| DNS Query                      | Browser requests connection       | HTTP GET Request (`GET /index.html`, `Host: www.example.com`) |
| **Transport Layer**            | **Transport Layer**               | **Transport Layer**                                           |
| UDP Port 53                    | TCP SYN → SYN-ACK → ACK           | TCP Port 80                                                   |
| **Internet Layer**             | **Internet Layer**                | **Internet Layer**                                            |
| IP packet to DNS Server        | IP packet to Web Server           | IP packet to Web Server                                       |
| **Network Access Layer**       | **Network Access Layer**          | **Network Access Layer**                                      |
| Ethernet Frame sent            | Ethernet Frame sent               | Ethernet Frame sent                                           |
| **Result:**                    | **Result:**                       | **Result:**                                                   |
| DNS returns IP                 | TCP connection established        | Server sends webpage                                          |

# Final Concept

The TCP/IP stack is used **three times** during web browsing:

```
1️⃣ DNS Query → Find server IP
2️⃣ TCP Handshake → Establish connection
3️⃣ HTTP Request → Request webpage
```

For **each stage** the data travels:

```
Application
     ↓
Transport
     ↓
Internet
     ↓
Network Access
```

And on the receiver:

```
Network Access
     ↑
Internet
     ↑
Transport
     ↑
Application
```


The data **travels through the TCP/IP model three separate times**:

```
1️⃣ DNS Query
Application → Transport → Internet → Network Access

2️⃣ TCP Handshake
Application → Transport → Internet → Network Access

3️⃣ HTTP Request
Application → Transport → Internet → Network Access
```
