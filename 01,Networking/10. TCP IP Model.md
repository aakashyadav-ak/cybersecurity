# TCP/IP Model
The TCP/IP Model (Transmission Control Protocol/Internet Protocol) is a practical, four-layer networking framework that forms the foundation of the modern internet.

```
┌─────────────────────────────────────┐
│         APPLICATION LAYER           │  Layer 4
├─────────────────────────────────────┤
│          TRANSPORT LAYER            │  Layer 3
├─────────────────────────────────────┤
│           INTERNET LAYER            │  Layer 2
├─────────────────────────────────────┤
│        NETWORK ACCESS LAYER         │  Layer 1
└─────────────────────────────────────┘
```

#  TCP/IP Model

##  What is the TCP/IP Model?
> **TCP/IP Model** (Transmission Control Protocol/Internet Protocol) is a **conceptual/Practical framework** used to describe network communication. It is the **practical implementation** model used in the **Internet** and most modern networks.

- Also known as: **Internet Protocol Suite** or **DoD Model** (Department of Defense)
- Developed by: **DARPA** in the 1970s
- More **practical** than OSI Model
- **De facto standard** for Internet communications

---

#  Detailed Layer Description

## Network Access Layer (Layer 1)

> **Function:** Defines how data is physically transmitted over the network medium

**Responsibilities:**
- Physical addressing (MAC addresses)
- Frame formatting
- Media access control
- Error detection (not correction)
- Physical transmission of bits

**Combines OSI Layers:**
- Physical Layer
- Data Link Layer

#### Protocols & Technologies:
- **Ethernet** (802.3)
- **WiFi** (802.11)
- **PPP** (Point-to-Point Protocol)
- **Frame Relay**
- **ATM** (Asynchronous Transfer Mode)
- **Token Ring**
- **HDLC** (High-Level Data Link Control)

#### Hardware:
- Network Interface Cards (NICs)
- Switches
- Bridges
- Hubs
- Cables (Copper, Fiber)
- Wireless Access Points

#### Data Unit:
- **Frame** (or **Bits** at physical level)

**Network Access Layer Diagram:**
```
┌─────────────────────────────────────┐
│  Ethernet Frame                     │
├──────────┬──────┬─────┬──────┬─────┤
│Preamble  │ MAC  │ MAC │ Data │ FCS │
│   (8)    │ Dest │ Src │      │ (4) │
└──────────┴──────┴─────┴──────┴─────┘

MAC Address: 48-bit (6 bytes)
Example: AA:BB:CC:DD:EE:FF
```

#### Physical Addressing (MAC Addresses)

At Layer 1, devices use MAC addresses to identify each other inside a local network (LAN).

A MAC address:

- 48-bit hardware address  
- Assigned to Network Interface Card (NIC)  
- Example:  
  ```
  00:1A:2B:3C:4D:5E
  ```

When a device sends data in a LAN:

- It uses **Source MAC**
- It uses **Destination MAC**

Switches forward frames using MAC addresses.

Example:

- Your Windows VM sends traffic  
- Switch forwards based on MAC table  

---

#### Frame Formatting

Layer 2 wraps data into a **Frame**.

Structure of a frame (Ethernet example):

```
| Destination MAC | Source MAC | Type | Data | FCS |         # This entire structure = Frame
```

**Trailer** = The last field of the frame
**FCS (Frame Check Sequence)** = The content inside the trailer

**Important protocol here:**  
Ethernet

So:

- Network layer sends a packet  
- Data Link layer encapsulates it into a frame  
- Adds MAC addresses + trailer  

---

#### Media Access Control (MAC)

If multiple devices share the same medium (like Ethernet cable or Wi-Fi), they need rules to avoid collision.

Ethernet uses:

- CSMA/CD (Carrier Sense Multiple Access with Collision Detection)

Wi-Fi uses:

- CSMA/CA (Collision Avoidance)

This ensures:

- Only one device transmits at a time  
- Reduces collision  

---

#### Error Detection (Not Correction)

Layer 2 detects errors using:

- FCS (Frame Check Sequence)

If a frame gets corrupted:

- Receiver detects error  
- Frame is dropped  
- No correction at this layer  

Correction happens at higher layers (like TCP).

---

#### Physical Transmission of Bits

This is **Layer 1 (Physical Layer)** responsibility.

Layer 1:

- Converts bits into electrical signals (Ethernet)  
- Light pulses (Fiber)  
- Radio waves (Wi-Fi)  

Protocols involved:

- IEEE 802.3  
- IEEE 802.11  

It handles:

- Voltage levels  
- Cable types  
- Bit timing  
- Connectors  

---

## Internet Layer (Layer 2)

> **Function:** Routes packets across networks using logical addressing (IP addresses)

**Responsibilities:**
- Logical addressing (IP addresses)
- Routing between networks
- Packet fragmentation and reassembly
- Path determination
- Best effort delivery (unreliable)

**Equivalent OSI Layer:**
- Network Layer

#### Key Protocols:

**IP (Internet Protocol):**
- **IPv4** - 32-bit addressing (e.g., 192.168.1.1)
- **IPv6** - 128-bit addressing (e.g., 2001:db8::1)

**Supporting Protocols:**
- **ICMP** (Internet Control Message Protocol) - ping, traceroute
- **ARP** (Address Resolution Protocol) - Maps IP to MAC
- **RARP** (Reverse ARP) - Maps MAC to IP
- **IGMP** (Internet Group Management Protocol) - Multicast
- **IPsec** - Security protocol

#### Routing Protocols:
- **RIP** (Routing Information Protocol)
- **OSPF** (Open Shortest Path First)
- **EIGRP** (Enhanced Interior Gateway Routing Protocol)
- **BGP** (Border Gateway Protocol)

#### Hardware:
- Routers
- Layer 3 Switches

#### Data Unit:
- **Packet** (or **Datagram**)


---

## Transport Layer (Layer 3)

> **Function:** Provides end-to-end communication, reliability, and flow control

**Responsibilities:**
- End-to-end communication
- Segmentation and reassembly
- Flow control
- Error detection and correction
- Port-based addressing
- Connection management

**Equivalent OSI Layer:**
- Transport Layer

#### Key Protocols:

**TCP (Transmission Control Protocol):**
- **Connection-oriented**
- **Reliable** delivery
- **Ordered** delivery
- **Flow control**
- **Error checking**
- Uses **3-way handshake**
- Use cases: HTTP, FTP, SSH, Email

**UDP (User Datagram Protocol):**
- **Connectionless**
- **Unreliable** (best effort)
- **Faster** than TCP
- **No flow control**
- **Minimal overhead**
- Use cases: DNS, DHCP, VoIP, Video streaming

#### Port Numbers:
- **Well-Known Ports:** 0-1023 (Reserved for common services)
- **Registered Ports:** 1024-49151 (Registered applications)
- **Dynamic/Private Ports:** 49152-65535 (Temporary/client ports)

#### Hardware:
- Firewalls
- Load Balancers

#### Data Unit:
- **Segment** (TCP) or **Datagram** (UDP)


#### TCP 3-Way Handshake:
```
Client                          Server
  |                               |
  |  SYN (Seq=100)               |
  |----------------------------->|
  |                               |
  |  SYN-ACK (Seq=200, Ack=101)  |
  |<-----------------------------|
  |                               |
  |  ACK (Seq=101, Ack=201)      |
  |----------------------------->|
  |                               |
  |  Connection Established       |
  |                               |
```

## Application Layer (Layer 4)
> **Function:** Provides network services directly to end-user applications

**Responsibilities:**
- Application-level protocols
- User interface
- Network process to application
- Data representation
- Session management
- Encryption (optional)

**Combines OSI Layers:**
- Application Layer
- Presentation Layer
- Session Layer

#### Common Protocols:

**Web & File Transfer:**
- **HTTP** (Hypertext Transfer Protocol) - Port 80
- **HTTPS** (HTTP Secure) - Port 443
- **FTP** (File Transfer Protocol) - Port 20, 21
- **TFTP** (Trivial FTP) - Port 69
- **SFTP** (SSH FTP) - Port 22

**Email:**
- **SMTP** (Simple Mail Transfer Protocol) - Port 25
- **POP3** (Post Office Protocol) - Port 110
- **IMAP** (Internet Message Access Protocol) - Port 143

**Network Services:**
- **DNS** (Domain Name System) - Port 53
- **DHCP** (Dynamic Host Configuration Protocol) - Port 67, 68
- **NTP** (Network Time Protocol) - Port 123
- **SNMP** (Simple Network Management Protocol) - Port 161, 162

**Remote Access:**
- **Telnet** - Port 23 (Insecure)
- **SSH** (Secure Shell) - Port 22
- **RDP** (Remote Desktop Protocol) - Port 3389
- **VNC** (Virtual Network Computing) - Port 5900

**Other:**
- **SMB** (Server Message Block) - Port 445
- **LDAP** (Lightweight Directory Access Protocol) - Port 389
- **SIP** (Session Initiation Protocol) - Port 5060

#### Data Unit:
- **Data** or **Message**


**HTTP Request Example:**

GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
Connection: keep-alive

## Data Encapsulation Process

┌─────────────────────────────────────────────────────┐
│ SENDER │
├─────────────────────────────────────────────────────┤
│ Application Layer: DATA │
│ ↓ │
│ Transport Layer: [TCP/UDP Header | DATA] │
│ ↓ │
│ Internet Layer: [IP Header | TCP | DATA] │
│ ↓ │
│ Network Access: [Frame Header | IP | TCP | DATA | FCS] │
│ ↓ │
│ Physical Bits │
│ ↓ │
│ Network Medium │
│ ↓ │
│ Physical Bits │
│ ↓ │
├─────────────────────────────────────────────────────┤
│ RECEIVER │
├─────────────────────────────────────────────────────┤
│ Network Access: [Frame Header | IP | TCP | DATA | FCS] │
│ ↓ (Remove Frame Header/FCS) │
│ Internet Layer: [IP Header | TCP | DATA] │
│ ↓ (Remove IP Header) │
│ Transport Layer: [TCP/UDP Header | DATA] │
│ ↓ (Remove TCP/UDP Header) │
│ Application Layer: DATA │
└─────────────────────────────────────────────────────┘

| **Internet**       | Packet           | Packet / Datagram              |
| **Network Access** | Frame            | Frame                          |
| **Physical**       | Bits             | Bits                           |

### Web Browsing (HTTP Request)

User Types: www.example.com

Step 1: DNS Resolution
┌─────────────────────────────────────────┐
│ Application: DNS Query │
│ Transport: UDP Port 53 │
│ Internet: IP to DNS Server │
│ Network Access: Ethernet Frame │

Step 2: TCP Connection (3-Way Handshake)
┌─────────────────────────────────────────┐
│ Client sends: SYN │

┌─────────────────────────────────────────┐
│ Application: GET /index.html HTTP/1.1 │

│ Network Access: Ethernet Frame │

Step 4: HTTP Response
┌─────────────────────────────────────────┐
│ Application: HTML Content │
│ Transport: TCP Port 80 │
│ Internet: IP from Server │
│ Network Access: Ethernet Frame │
└─────────────────────────────────────────┘

Step 5: TCP Connection Termination (4-Way)
┌─────────────────────────────────────────┐
│ Client: FIN │
│ Server: ACK │
