WebSocket is a protocol that provides full-duplex (two-way) communication channels over a single TCP connection. Unlike HTTP, the connection stays open, allowing the server to push data to the client instantly.

WebSocket is a communication protocol that provides: 
- Full-duplex (two-way) communication
- Persistent connection between client and server 
- Real-time data transfer
- Low latency messaging
 
  **Used for:** Chat apps, Gaming, Live updates, Stock tickers, Notifications

##  1: Cross-Site WebSocket Hijacking (CSWSH)

Cross-Site WebSocket Hijacking is the WebSocket equivalent of CSRF (Cross-Site Request Forgery). It occurs when an attacker's website can establish a WebSocket connection to a vulnerable application, effectively hijacking the victim's authenticated session.

**Vulnerability:**

When a WebSocket handshake occurs:
- Browser automatically sends cookies (including session cookies)
- The Sec-WebSocket-Key header is NOT a security feature (it's just for proxy detection)
- If server doesn't validate the Origin header, any website can connect

## 6.Broken Access Control
WebSocket applications often fail to implement proper authorization checks for each message/action.

In traditional HTTP applications, every request goes through authentication and authorization checks. However, in WebSocket applications, developers often make a critical mistake: they authenticate users only when the connection is established, then trust all subsequent messages without verification.

```
Server returns User B's profile â† Access control failure
```
